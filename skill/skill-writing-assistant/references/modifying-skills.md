## 工作流：修改现有技能

### 第1步：诊断问题

使用以下清单识别问题：

**技能不上传：**
- 错误："在上传的文件夹中找不到SKILL.md"
- 原因：文件未精确命名为SKILL.md
- 解决方案：重命名为SKILL.md（区分大小写）

**技能不触发：**
- 症状：技能从不自动加载
- 解决方案：修订您的描述字段
- 快速检查清单：
  - 是否太泛泛？（"帮助处理项目"无效）
  - 是否包含用户实际会说的触发短语？
  - 如果适用，是否提及相关文件类型？

**调试方法：**
询问Claude："您何时会使用[技能名称]技能？"Claude将引用描述。根据缺少的内容进行调整。

**技能触发太频繁：**
- 症状：技能为不相关查询加载
- 解决方案：
  1. 添加负面触发：
     ```yaml
     description: CSV文件的高级数据分析。用于统计建模、回归、聚类。不用于简单数据探索（使用data-viz技能代替）。
     ```
  2. 更具体：
     ```yaml
     # 太宽泛
     description: 处理文档
     
     # 更具体
     description: 处理用于合同审查的PDF法律文档
     ```
  3. 澄清范围：
     ```yaml
     description: 用于电子商务的PayFlow支付处理。专门用于在线支付工作流，不用于一般财务查询。
     ```

**MCP连接问题：**
- 症状：技能加载但MCP调用失败
- 检查清单：
  1. 验证MCP服务器是否连接
  2. 检查身份验证（API密钥、权限、OAuth令牌）
  3. 独立测试MCP（不通过技能直接调用）
  4. 验证工具名称（技能引用正确的MCP工具名称）

**指令不被遵循：**
- 症状：技能加载但Claude不遵循指令
- 常见原因：
  1. 指令太冗长 - 保持简洁，使用项目符号
  2. 指令被埋没 - 将关键指令放在顶部，使用`##重要`标题
  3. 模糊语言 - 使用具体、可操作的指令
  4. 模型"懒惰" - 在用户提示中添加明确鼓励

**大上下文问题：**
- 症状：技能似乎缓慢或响应质量下降
- 原因：
  - 技能内容太大
  - 同时启用了太多技能
  - 所有内容都加载了，而不是渐进式披露
- 解决方案：
  1. 优化SKILL.md大小（移动到references/，保持SKILL.md在5,000字以下）
  2. 减少启用的技能数量（评估是否同时启用了20-50个技能）

### 第2步：应用改进模式

根据文档第5章，应用以下模式：

**模式1：顺序工作流编排**
- 使用时机：用户需要按特定顺序的多步骤流程
- 关键技术：明确的步骤顺序、步骤间依赖关系、每个阶段的验证、失败的回滚指令

**模式2：多MCP协调**
- 使用时机：工作流跨越多个服务
- 关键技术：清晰的阶段分离、MCP间的数据传递、进入下一阶段前的验证、集中错误处理

**模式3：迭代优化**
- 使用时机：输出质量随迭代提高
- 关键技术：明确的质量标准、迭代改进、验证脚本、知道何时停止迭代

**模式4：上下文感知工具选择**
- 使用时机：相同结果，根据上下文使用不同工具
- 关键技术：清晰的决策标准、备用选项、关于选择的透明度

**模式5：领域特定智能**
- 使用时机：您的技能增加了超越工具访问的专门知识
- 关键技术：逻辑中嵌入的领域专业知识、行动前的合规性检查、全面文档、清晰治理

### 第3步：测试修改

1. 运行触发测试，确保修改没有破坏现有功能
2. 执行功能测试，验证改进是否有效
3. 进行性能比较，证明修改带来了改进
4. 收集用户反馈，进行进一步迭代
