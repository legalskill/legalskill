# 项目检查清单

## 概述
本检查清单用于确保新项目设置完整且符合最佳实践。

## 项目初始化检查

### 基础结构
- [ ] 项目文件夹使用kebab-case命名（如 `my-project`）
- [ ] 包含清晰的README.md文件
- [ ] 设置适当的.gitignore文件
- [ ] 初始化Git仓库
- [ ] 创建初始提交

### 文档
- [ ] README包含项目描述和用途
- [ ] README包含安装和运行说明
- [ ] README包含贡献指南
- [ ] 如有必要，创建API文档
- [ ] 创建CHANGELOG.md记录版本变更

## 代码质量检查

### 代码规范
- [ ] 设置代码格式化工具（Prettier、Black等）
- [ ] 配置代码检查工具（ESLint、Pylint等）
- [ ] 统一缩进和代码风格
- [ ] 删除未使用的导入和代码
- [ ] 添加必要的注释

### 测试
- [ ] 配置测试框架
- [ ] 编写单元测试
- [ ] 编写集成测试
- [ ] 设置测试覆盖率检查
- [ ] 创建测试数据

## 安全检查

### 敏感信息
- [ ] 检查代码中是否有硬编码的API密钥
- [ ] 将敏感配置移至环境变量
- [ ] 使用.gitignore排除配置文件
- [ ] 审查依赖项的安全性

### 权限
- [ ] 验证文件权限设置
- [ ] 确认数据库访问权限
- [ ] 检查API端点权限控制
- [ ] 审核第三方服务权限

## 部署检查

### 构建配置
- [ ] 配置构建脚本或CI/CD
- [ ] 设置版本控制策略
- [ ] 创建部署清单
- [ ] 配置监控和日志

### 环境配置
- [ ] 为不同环境设置配置（开发、测试、生产）
- [ ] 创建环境变量文档
- [ ] 配置健康检查端点
- [ ] 设置备份策略

## 团队协作检查

### 开发流程
- [ ] 设置分支策略
- [ ] 定义Pull Request模板
- [ ] 配置代码审查规则
- [ ] 确定版本发布流程

### 沟通
- [ ] 创建项目沟通渠道
- [ ] 设置问题跟踪流程
- [ ] 定义紧急响应流程
- [ ] 建立知识共享机制

## 维护检查

### 监控
- [ ] 设置错误监控
- [ ] 配置性能监控
- [ ] 创建报警规则
- [ ] 设置日志聚合

### 更新
- [ ] 制定依赖更新策略
- [ ] 规划技术债务处理
- [ ] 设置定期安全审计
- [ ] 创建应急预案

## 技能特定检查（Claude Skills）

### 技能结构
- [ ] 文件夹名称为kebab-case
- [ ] SKILL.md文件存在且命名正确
- [ ] Frontmatter格式正确
- [ ] name字段与文件夹名匹配

### 技能内容
- [ ] description字段包含功能和触发条件
- [ ] 指令清晰且可操作
- [ ] 包含错误处理
- [ ] 提供使用示例

### 技能测试
- [ ] 在明显任务上测试触发
- [ ] 在转述请求上测试触发
- [ ] 验证无关话题不触发
- [ ] 功能测试通过

## 发布检查

### 最终验证
- [ ] 所有测试通过
- [ ] 代码审查完成
- [ ] 文档完整且准确
- [ ] 性能符合要求

### 分发准备
- [ ] 压缩为.zip文件
- [ ] 创建安装说明
- [ ] 准备发布说明
- [ ] 更新版本号

---

## 重要提醒

1. **安全第一**：始终优先考虑安全性
2. **渐进开发**：从核心功能开始，逐步添加
3. **文档同步**：代码变更时更新文档
4. **团队沟通**：保持信息透明和共享
5. **定期回顾**：定期回顾和优化流程

---

*本检查清单会根据项目需求和技术发展定期更新。建议每个项目开始前和执行关键阶段时使用此清单。*
