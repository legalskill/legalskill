# API 模式参考指南

## 概述
本文件提供使用MCP（Model Context Protocol）和Claude Skills时的API最佳实践和模式。

## 核心模式

### 1. 速率限制处理
- **模式**：指数退避重试
- **场景**：应对API速率限制和临时故障
- **实现**：重试时等待时间指数级增加，最多重试3次

### 2. 错误处理
- **模式**：优雅降级
- **场景**：当依赖服务不可用时
- **实现**：提供简化功能或缓存数据作为后备

### 3. 数据验证
- **模式**：提前验证
- **场景**：处理用户输入数据
- **实现**：在执行API调用前验证所有输入参数

## 最佳实践

### 安全实践
- 永远不要将API密钥嵌入代码中
- 使用环境变量或安全存储
- 为不同环境使用不同的密钥

### 性能优化
- 批量处理请求以减少API调用次数
- 使用缓存减少重复请求
- 监控API响应时间并设置超时

## 故障排除

### 常见错误
1. **429 Too Many Requests** - 速率限制触发，等待后重试
2. **401 Unauthorized** - API密钥无效或过期
3. **403 Forbidden** - 权限不足或令牌范围错误
4. **500 Internal Server Error** - 服务端错误，重试或联系支持

### 调试技巧
- 启用详细日志记录
- 记录完整的请求/响应信息
- 使用API调试工具（如Postman）测试独立调用
