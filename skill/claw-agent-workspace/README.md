# Claw 工作区管理

> 面向人类维护者的技能说明文件

## 概述

Claw 是一个嵌入式智能体运行时，每个网关对应一个智能体进程。本技能动态管理 Claw 系列智能体（含 OpenClaw 及同级别 Claw Agent 实现）的工作区引导文件。

核心能力：扫描绑定技能目录 → 提取关键词和模块路由 → 生成动态"核心技能触发块"注入 AGENTS.md——任何 agent 通过升级即可自动获得"关键词检测→强读技能文件→模块路由"机制。

## 关于本技能

本技能指导你创建和修改 Claw 智能体的主配置文件和工作区引导文件：

1. 创建和修改主配置文件（JSON5 格式）
2. 创建和修改工作区引导文件（AGENTS.md、SOUL.md 等）
3. 使用内置默认模板快速启动
4. 单技能工作区模式管理和注册
5. 智能体升级与技能感知叠加

## 使用场景

- 配置 Claw 智能体的工作区、端口、频道等基础参数
- 创建/修改智能体引导文件（AGENTS.md、SOUL.md 等）
- 升级 agent 同步最新技能路由
- 管理工作区备份与回滚
- 单技能场景下的技能注册与触发
- 检查工作区文件完整性和一致性

## 核心特性

### 分级修改模式

| 模式 | 触发条件 | 行为 | 风险 |
|------|----------|------|------|
| 标准模式 (L1) | 默认，无"升级"关键词 | 仅修复格式规范，不覆盖受保护文件 | 极低 |
| 深度模式 (L2) | 用户明确要求"升级 agent" | 自动备份后扫描技能目录，重新生成核心技能触发块，应用最新模板 | 中高 |

### 单技能模式

当用户要求"单技能"时，本技能将工作区切换到单技能模式：禁用多技能路由表扫描，Agent 直读该技能文件，同时在 AGENTS.md 中生成动态路由块。

### 受保护文件

IDENTITY.md、USER.md、MEMORY.md 默认受保护，修改时不会直接覆盖而保留用户已积累的内容。

## 设计与贡献

本技能的设计遵循以下原则：

- **最小修改原则**：L1 模式不改逻辑，L2 模式先备份再操作
- **动态路由感知**：核心技能触发块每次从实际技能目录重新扫描生成
- **备份可回滚**：L2 操作自动产生 `.bak-*` 备份文件，紧急情况下可手动还原
