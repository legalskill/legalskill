# 贡献指南

感谢你愿意为律锥·legalskill（法律技能开源社区）贡献！在提交前，请了解我们的核心原则和流程。

## 核心原则

1. **实战导向**：技能必须源于真实法律工作场景，拒绝纯粹秀技术。
2. **合规为底**：所有技能输出必须包含免责声明；涉及数据检索的，需注明来源与可溯源性。
3. **开源透明**：技能文件不加密、不隐藏后门，确保使用者可审查。
4. **双许可兼容**：代码使用 Apache 2.0，内容使用 CC BY-SA 4.0。

## 贡献方式

### 方式一：提交新技能（PR）

1. Fork 本仓库。
2. 在 `skill/` 目录下新建技能文件夹，参考 [legal-consultation](./skill/legal-consultation) 的目录结构。
3. 确保包含以下文件：
   - `SKILL.md`（头部标注 `license: CC BY-SA 4.0`，末尾含完整免责声明）
   - `skill.yaml`（标准化元数据）
   - 如有代码，文件头部标注 `# SPDX-License-Identifier: Apache-2.0`
4. 提交 PR，等待审核。

### 方式二：推荐外部技能

如果你发现优秀的法律技能仓库，希望在律锥社区展示：

1. 在 [GitHub Discussions - 技能推荐](https://github.com/legalskill/legalskill/discussions/categories/ideas) 发帖。
2. 提供技能仓库链接和推荐理由。
3. 维护者审核后，将其收录至 [技能目录](./SKILL_CATALOG.md)。

### 方式三：反馈与讨论

- 使用问题或功能建议：提交 Issue。
- 一般性讨论：在 GitHub Discussions 发帖。

## 技能审核标准

提交的技能需满足以下条件：

- [ ] 包含完整的 `SKILL.md` 和 `skill.yaml`
- [ ] `SKILL.md` 末尾包含免责声明
- [ ] 明确标注适用的许可
- [ ] 法律逻辑清晰，无明显专业错误
- [ ] 如涉及外部 API 或数据源，已说明来源

## 社区行为准则

- 尊重每一位贡献者和使用者。
- 就事论事，聚焦技能本身。
- 法律意见类讨论请注明“仅供参考，不构成法律意见”。
